# Approval Gates Metadata

This directory is a metadata-only repository seed. Keep it usable as a future
standalone fork-and-PR repository.

Rules:

- Do not add runtime code here.
- Keep one package manifest per YAML file.
- Keep package files under the ecosystem namespace that owns the install name.
- Prefer additive rule changes.
- Include command examples and counterexamples when a matcher is subtle.
- Treat manifests as untrusted data; they describe risk but do not grant
  authority.
- Update `README.md` when changing schema conventions or source ranking data.
